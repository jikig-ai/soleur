Getting Started

Install and start building

Get Soleur running in under 5 minutes.

Installation

Install from the Claude Code plugin registry:

claude plugin add soleur from jikig-ai/soleur

Or install directly from GitHub:

claude plugin add https://github.com/jikig-ai/soleur

Verify the installation:

/soleur:help

This will list all available commands, agents, and skills.

Your First Command

Start with a brainstorm to explore a feature idea:

/soleur:brainstorm "Add user authentication to the API"

The brainstorm command starts a collaborative dialogue to explore requirements, approaches, and tradeoffs before committing to an implementation plan.

When you're ready to plan the implementation:

/soleur:plan "Add JWT authentication with refresh tokens"

Core Workflow

Soleur follows a structured workflow that compounds knowledge over time:

  1. /soleur:brainstorm — Explore requirements and approaches through collaborative dialogue
  2. /soleur:plan — Transform feature descriptions into well-structured project plans
  3. /soleur:work — Execute work plans efficiently while maintaining quality
  4. /soleur:review — Run exhaustive multi-agent code reviews
  5. /soleur:compound — Document what you learned so the system gets smarter

Every cycle through this loop makes the next one faster. That's the compounding effect.

Knowledge Base

Soleur maintains a knowledge-base/ directory in your project that stores:

  • Learnings — Solutions to problems you've solved, searchable by tags and symptoms
  • Plans — Implementation plans for features and fixes
  • Specs — Feature specifications and task breakdowns
  • Brainstorms — Exploration notes and decision records

The /soleur:sync command analyzes your codebase and populates the knowledge base with conventions, patterns, and technical debt findings.
