# feat: X/Twitter Account Provisioning via Ops-Provisioner

## Overview

Provision the X/Twitter account, Developer Portal, and API keys using the ops-provisioner agent's guided workflow. The X integration code (PR #466) is complete but inert -- all scripts exist, no credentials do. This is the third use of ops-provisioner (after Cloudflare and Plausible).

Closes #474.

## Problem Statement / Motivation

The community-manager agent has X/Twitter code (`x-community.sh`, `x-setup.sh`) that cannot be tested or used because no X account exists. Two verification tasks in `knowledge-base/project/specs/feat-community-agent-x/tasks.md` are blocked on account registration.

## Proposed Solution

Fix the `.env` path resolution bug in both `x-setup.sh` and `discord-setup.sh`, then invoke the ops-provisioner with X as the target tool.

### Code Fix (Pre-provisioning)

**`.env` path resolution bug:** Both `x-setup.sh` and `discord-setup.sh` use `local env_file=".env"` (relative path). When run from a worktree, credentials land in the wrong directory. Fix: resolve via `git rev-parse --show-toplevel` so `.env` is always at repo root.

### Provisioning Flow

1. **Pre-check:** Founder manually verifies `@soleur` availability at `x.com/soleur` (fallback: `@soleur_ai`)
2. **Account registration:** Invoke ops-provisioner with signup URL `https://x.com/i/flow/signup`. Founder completes entire signup manually.
3. **Developer Portal + API keys:** Ops-provisioner navigates to `developer.x.com`. Founder applies for developer access, creates project/app, configures OAuth 1.0a (callback: `https://soleur.ai`, permissions: Read+Write), generates keys. If approval requires review, pause and re-invoke when approved.
4. **Credential storage:** Founder sets env vars inline and runs `x-setup.sh write-env`, then `x-setup.sh verify`.
5. **Expense recording:** Ops-provisioner records X API in expense ledger.

## Technical Considerations

- **Worktree path:** Both `x-setup.sh` and `discord-setup.sh` need the same fix -- resolve `.env` to repo root. Fixes all worktree contexts.
- **Credential handoff:** Founder exports env vars inline (`X_API_KEY=xxx ... bash x-setup.sh write-env`) and runs `verify`. No new commands needed.
- **Developer Portal approval delay:** If not instant, provisioner pauses. Founder re-invokes when approved.

## Acceptance Criteria

- [ ] `.env` path uses `git rev-parse --show-toplevel` in both `x-setup.sh` and `discord-setup.sh`
- [ ] X account registered with preferred handle
- [ ] All 4 env vars validated via `x-setup.sh verify`
- [ ] Expense ledger updated with X API entry
