# Plan globs must be verified against repo structure before implementing

## Problem

PR #2889 added a new preflight Check 5 ("Brand-Survival Self-Review") whose
function is gating ship-time on PRs that touch sensitive paths. The plan
prescribed a literal glob list:

```
- apps/web-platform/server/**
- apps/web-platform/supabase/**
- apps/web-platform/lib/stripe*
- apps/web-platform/lib/auth*
- apps/web-platform/lib/byok*
- infra/**
- **/doppler*.{yml,yaml,sh}
- .github/workflows/*doppler*.yml
```

I implemented this verbatim in `plugins/soleur/skills/preflight/SKILL.md`. The
diff was lint-clean and tests passed. Multi-agent review found three P1s:

1. **Missing every Next.js attack surface outside `server/**`** —
   `middleware.ts` (CSP, route guards, auth cookies), `app/api/**` (every HTTP
   handler including `auth/`, `billing/`, `webhooks/`, `keys/`), and
   `apps/web-platform/lib/{security-headers,csp,log-sanitize,safe-session,safe-return-to,supabase,legal,auth}*`
   were silently absent. A PR that swaps auth middleware, weakens CSP, or
   modifies a billing API route would skip Check 5 entirely (return SKIP "No
   sensitive paths touched") — defeating the whole #2887 ship-time gate.

2. **`infra/**` is a non-existent root path** — the actual Terraform root is
   `apps/web-platform/infra/**`. Every PR touching production Terraform
   silently bypassed Check 5 because the glob never matched anything.

3. **`.github/workflows/*doppler*.yml` matched zero current workflows** — none
   of the deploy/release/secret-handling workflows include `doppler` in the
   filename, even though they consume `DOPPLER_TOKEN_*`.

The plan's glob list "looked right" because each line described a category
that does exist in the repo — but the *paths* were wrong, the *coverage* was
incomplete, and the failure mode was silent (SKIP, not FAIL).

## Solution

Replaced the bullet-list globs with a canonical regex (single source of truth
mirrored verbatim into deepen-plan Phase 4.6) covering every concrete attack
surface I could enumerate against the actual repo:

```bash
SENSITIVE_PATH_RE='^(apps/web-platform/(server|supabase|app/api|middleware\.ts$)|apps/web-platform/lib/(stripe|auth|byok|security-headers|csp|log-sanitize|safe-session|safe-return-to|supabase)|apps/web-platform/lib/(legal|auth)/|apps/[^/]+/infra/|.+/doppler[^/]*\.(yml|yaml|sh)$|\.github/workflows/.*(doppler|secret|token|deploy|release|version-bump|web-platform|infra-validation|cla|cf-token|linkedin-token).*\.ya?ml$)'
```

Verification: the regex was generated by enumerating actual files via
`git ls-files apps/web-platform/`, `git ls-files apps/web-platform/lib/`,
`git ls-files .github/workflows/`, and reading each candidate against the
review agents' attack-surface inventory.

Commit: `53afac01` (PR #2889).

## Key Insight

**A plan that prescribes a path glob is a falsifiable claim about the repo,
not a placeholder.** Implementing the glob without verifying the claim
holds is the same class of bug as implementing a relative-path `../`
without tracing it (rule `hr-when-a-plan-specifies-relative-paths-e-g`).
The failure mode is identical — silent miss at runtime — and the fix is
identical: verify every prescribed path token against live repo state
before committing.

This is especially load-bearing for *negative-coverage* gates (security
gates, denylist filters, sensitive-path detectors) where a glob that
matches "almost the right thing" silently fails open. Compare to
*positive-coverage* gates (test runners, build inputs) where a missed
file usually produces a loud error at runtime.

## Generalization

When a plan prescribes any of the following, verify against live repo state
before implementing:

| Plan token | Verification command |
|---|---|
| Relative path (`../foo`) | trace each `../` step; `ls` the final target |
| Absolute path (`apps/X/Y/foo`) | `ls apps/X/Y/foo` (file exists) |
| Path glob (`apps/X/**`) | `git ls-files \| grep -E '<translated-glob>'` (≥1 match) |
| Workflow filename (`*foo*.yml`) | `ls .github/workflows/*foo*.yml` (≥1 match) |
| Symbol or function name | `rg <symbol>` (≥1 hit) |
| Phase/check identifier (`Phase 4.6`, `Check 4`) | `grep -n '^### Phase\|^### Check' <target>` (no collision) |

For *negative-coverage* gates, also enumerate sibling architectural-layer
files at the same depth — e.g., for `apps/web-platform/lib/auth*`, also
inspect `apps/web-platform/lib/{security-headers,csp,session,...}.ts` and
ask: "if my gate is supposed to catch credential-handling, does this
sibling also handle credentials?" Globs constructed by transcribing the
plan miss files the plan never inventoried.

## Session Errors

- **Phase numbering collision (preflight Check 4 → Check 5).** Plan
  prescribed Check 4 but main had already added Check 4 (Environment
  Isolation from #2887) before this branch was created. Caught after
  merging main and re-reading preflight/SKILL.md. **Recovery:** renumbered
  to Check 5; updated the AGENTS.md `[skill-enforced: …]` annotation to
  match. **Prevention:** when a plan prescribes a numbered phase/section
  identifier, run `grep -n '^### Check\|^### Phase' <target-file>` AFTER
  merging main and BEFORE drafting any cross-reference that cites the
  number — plans authored against an older base may collide with newer
  additions.

- **Glob paths in plan implemented without verifying repo structure.** Plan
  said `infra/**` and `.github/workflows/*doppler*.yml`; both match zero
  real paths. Plan also missed `middleware.ts`, `app/api/**`, and several
  `lib/*` security/auth/session modules. Review caught 3 P1s before merge.
  **Recovery:** rewrote as canonical regex covering all attack surfaces
  (commit 53afac01). **Prevention:** see Generalization table above —
  treat globs as falsifiable claims, not placeholders.

- **CWD persistence confusion.** Ran `cd plugins/soleur && bun test ...`
  and the next bare-path Bash call resolved relative to `plugins/soleur`
  instead of worktree root. **Recovery:** switched to absolute paths.
  **Prevention:** rule `cq-for-local-verification-of-apps-doppler`
  already covers this — chain `cd <abs-path> && cmd` in a single call.
  I followed it for the test invocation but forgot for follow-up grep
  commands.

- **Non-canonical token-budget metric cited.** I cited "2681 words" for
  cumulative agent descriptions using
  `grep -h 'description:' agents/**/*.md` (non-recursive); the plugin
  AGENTS.md prescribes `shopt -s globstar && grep -h ...` (recursive)
  which gives 1508 words. Reviewer caught the false alarm. **Recovery:**
  marked as wontfix in synthesis. **Prevention:** use the prescribed
  command verbatim from `plugins/soleur/AGENTS.md` Token Budget Check;
  don't substitute approximations of metric tooling.
