1. **user-impact F1/F2/F3 — Light mode breaks credential/billing surfaces.** 53 untokenized files (#3279) include `(auth)/login`, `signup`, `setup-key`, `accept-terms`, `settings/billing-section`, `settings/cancel-retention-modal`, `settings/key-rotation-form`, `settings/api-usage-*`, `concurrency/upgrade-at-capacity-modal`, `chat/conversations-rail`. **Decision pending — see "Open decision" below.**
2. **code-quality F1 — Hydration flicker.** ThemeProvider initializes `theme="system"` then overwrites with stored value via two effects. Fix: `useState(() => readStoredTheme())` lazy initializer + `suppressHydrationWarning` on `<html>` in `app/layout.tsx`.
3. **architecture P1 #1 — Tailwind `dark:` variant divergence.** `components/settings/api-usage-section.tsx` (~19 lines), `api-usage-info-tooltip.tsx`, `api-usage-retry-button.tsx` use `dark:bg-zinc-900` etc. Tailwind v4 default resolves via `prefers-color-scheme`; diverges from new `data-theme` contract. Fix inline in `app/globals.css`:
   ```css
   @custom-variant dark {
     &:where([data-theme=dark], [data-theme=dark] *) { @slot; }
     @media (prefers-color-scheme: dark) {
       &:where([data-theme=system], [data-theme=system] *) { @slot; }
     }
   }
   ```
4. **user-impact F7 — Missing CSP regression test.** Spec acceptance criterion line 88 requires it. Add `apps/web-platform/test/theme-csp-regression.test.ts(x)` asserting `<NoFoucScript nonce={"test-nonce"}>` renders `<script nonce="test-nonce">`.
