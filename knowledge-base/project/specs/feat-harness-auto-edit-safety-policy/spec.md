# Spec: Harness Auto-Edit Safety Policy — Semantic-Weakening Detector + Additive-Only Boundary

## Problem Statement

Soleur's self-improvement loop (Self-Harness Layer 2) will eventually auto-propose edits to its own
operating rules (`AGENTS.md` hard rules + skills). The build is soak-gated (#6038 criterion 1 needs
≥1 month of #6037 weakness-digests; cannot clear before ~2026-08-05). But its most dangerous failure
mode is already reachable **today** by a human or any other bot: an edit that silently **narrows or
weakens** an existing `hr-*` guardrail (e.g. "Never X" → "avoid X where practical") while changing no
rule ID and removing no line — passing every existing gate.

Verified gap: `cron-compound-promote.ts:216` (`diffRemovesHardRule`) is a line-removal regex scoped
to `AGENTS.core.md`; `scripts/lint-rule-ids.py` is ID-based only; ADR-069 eval-gate measures
skill-arm fixture pass-rate, not guardrail coverage. Nothing baselines or diffs rule-body *strength*.
The `cq-rule-ids-are-immutable` gap ("must be closed first" per #6038) is exactly this: it blocks ID
reuse, not semantic weakening.

## Goals

- G1: Close the semantic-weakening gap with a landed CI gate that fails closed.
- G2: Define the additive-only boundary for future harness self-edits in a Provisional ADR (ADR-092).
- G3: Satisfy #6038 re-evaluation criteria 2 (ADR) and 3 (owner) without doing the soak-gated build.
- G4: Preserve the recursion invariant — harness-guardrail code stays outside the auto-editable set.

## Non-Goals

- NG1: The auto-proposer BUILD itself (stays deferred under #6038; soak-gated).
- NG2: Extending `cron-compound-promote.ts`'s `diffRemovesHardRule` / broadening its scope now —
  deferred to the build so its requirements are informed by real digest evidence (CPO caution).
- NG3: An LLM-judge as a *gating* signal — advisory only in v1.
- NG4: Tuned lexer thresholds — kept provisional, calibrated during the #6037 soak.

## Functional Requirements

- FR1: **Body-hash manifest.** A committed artifact (lean: `.claude/rule-body-hashes.txt`) mapping
  each `hr-*`/`wg-*` rule id → `sha256(normalized single-line body)`, regenerated by a script and
  CI-verified. A stale/missing manifest fails closed.
- FR2: **Weakening lexer.** A deterministic check flags a deontic-strength downgrade
  (`never|must not|always|MUST` → `avoid|should|prefer|where practical|when feasible`), hedge-word
  insertion, net token deletion, and quantifier/enumerated-case removal in an `hr-*`/`wg-*` body.
  `cq-regex-unicode-separators-escape-only` applies.
- FR3: **Body-change gate.** For any rule id present in both `HEAD` and the working copy whose body
  line changed: require a matching hash-manifest bump in the same commit AND a lexer pass. A lexer
  weakening-hit blocks pending explicit human ack.
- FR4: **Compliance-control escalation.** Any diff touching a rule tagged `[compliance-tier]` (reuse
  the existing inline marker) auto-flags mandatory human review — never auto-flow, even additively.
- FR5: **Additive-only policy (ADR-092, Provisional).** Codify: new rule / new skill section = eligible
  for auto draft-PR; any edit to an existing `hr-*`/`wg-*` body = human-only; any rule deletion =
  human-only hard block. The safe primitive is "add a rule," never "revise a rule."
- FR6: **Owner as CI gate + ack.** Resolve #6038 criterion 3 as a required CI check (FR3) plus a rule
  that any harness self-edit needs explicit operator ack — not a named org role.
- FR7: **Recursion invariant, pinned.** ADR-092 documents (and a test asserts) that the allowlist,
  hash-manifest, and detector code remain outside the auto-editable target set.

## Technical Requirements

- TR1: CI re-derives ALL gating values itself; never trusts proposer/LLM-supplied hashes or diffs
  (per the 2026-05-11 compound-promote P1 post-mortem — "hash covers field A, decision depends on B").
- TR2: Detector home: sibling `scripts/lint-rule-bodies.py` wired into the same lefthook + CI path as
  `lint-rule-ids.py`, keeping ID-immutability and body-weakening as separable concerns.
- TR3: Hashing normalization must ignore no-op reformatting (trailing whitespace, tag ordering) but
  not mask a weakening.
- TR4: Guardrail-count / body-hash invariant: a bot-authored PR can never decrease the rule count or
  change an `hr-*`/`wg-*` body hash without human ack.
- TR5: ADR-092 authored via `/soleur:architecture create` at plan time (Provisional; named revisit
  trigger = first #6037 digest with N samples).
- TR6: `cq-write-failing-tests-before` — the lexer + gate ship with failing-first tests covering a
  known weakening (strength downgrade), a benign additive edit, and a no-op reformat.

## Acceptance Criteria

- AC1: A crafted diff weakening an `hr-*` body under a stable ID is BLOCKED by CI with a clear message.
- AC2: A benign additive edit (new rule, or stronger enumerated case) PASSES.
- AC3: A no-op reformat of a rule body does NOT trip the gate.
- AC4: A diff touching a `[compliance-tier]` rule is flagged for mandatory human review.
- AC5: ADR-092 exists, is marked Provisional, and states the additive-only boundary + revisit trigger.
- AC6: #6038 is updated linking these artifacts as satisfying criteria 2 + 3; the build stays deferred.

## Out-of-Scope / Deferred (tracked under #6038)

The full eval-gated auto-proposer (mining → additive-only proposal → validation → draft PR) remains
deferred under #6038 until the soak clears (~2026-08-05) and ADR-092's additive-only boundary +
detector (this spec) are in place.
