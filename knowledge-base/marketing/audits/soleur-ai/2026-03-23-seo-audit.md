## Critical Issues

### 1. No canonical URL, OG tags, or Twitter cards rendered in production HTML (HIGH)

**Source template has them; production output appears to strip them.**

The `base.njk` template (line 7-23) correctly defines `<link rel="canonical">`, all OG tags (`og:title`, `og:description`, `og:url`, `og:type`, `og:site_name`, `og:locale`, `og:image`), and Twitter card tags (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`, `twitter:site`, `twitter:creator`).

However, fetching the live homepage at `https://soleur.ai` shows **none of these tags present** in the rendered HTML. The blog post at `/blog/what-is-company-as-a-service/` also shows all 12 tags as MISSING in rendered output.

Possible causes:
- The `<base href="/">` tag on line 64 of `base.njk` could interfere with relative URL resolution but should not strip meta tags
- A CDN/edge worker or HTML minifier may be stripping meta tags
- The Eleventy build may have a template rendering issue where variables are undefined, producing empty tags that get collapsed

**Impact:** Without OG tags, social sharing on Facebook, LinkedIn, Twitter/X, and Slack will show generic previews instead of rich cards. Without canonical URLs, search engines may index duplicate pages (e.g., trailing-slash variants). This is the single most impactful SEO issue on the site.

**Files:** `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk` (lines 7-23)

**Verification needed:** Build the site locally with `npx @11ty/eleventy` and inspect `_site/index.html` to confirm whether the tags are present in the static output. If they are, the issue is at the CDN/hosting layer. If they are not, the issue is in the Eleventy build.

### 3. Author URL points to homepage, not a dedicated author profile (MEDIUM-HIGH)

**File:** `/home/runner/work/soleur/soleur/plugins/soleur/docs/_data/site.json` (line 27)

The `site.author.url` is set to `"https://soleur.ai"` (the homepage), not an author page or bio. In the blog post template (`blog-post.njk`, line 39), the author link `<a href="{{ site.author.url }}" rel="author">{{ site.author.name }}</a>` links to the homepage.

For E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness), Google expects author attribution to link to a page that establishes the author's credentials. The JSON-LD `BlogPosting` schema also uses this URL for the author, meaning structured data and visible content both point to a non-author page.

**Impact:** Weakens E-E-A-T signals for blog content. Google's helpful content system evaluates whether content demonstrates first-hand expertise. An author link that resolves to a product homepage rather than a person's bio or about page undermines this signal.

**Recommendation:** Create an `/about/` or `/team/jean-jikig/` page with founder bio, credentials, and links to social profiles. Update `site.json` author URL to point there.

### 4. No dedicated About page exists (MEDIUM-HIGH)

No `/about/`, `/about.html`, or `/pages/about.html` page exists on the site. The footer and navigation link to Getting Started, Blog, Community, Vision, and Legal -- but no About page.

### 5. Atom feed contains only 6 of 10 blog posts (MEDIUM)

The site has 10 blog posts in the sitemap, but the Atom feed at `/blog/feed.xml` contains only 6 entries. The `eleventy.config.js` feed plugin configuration sets `limit: 20` (line 12), so the limit is not the issue.

Missing from feed (present in sitemap):

- `case-study-business-validation`
- `case-study-legal-document-generation`
- `case-study-operations-management`

These are all case study posts. If they lack the `blog` tag in their frontmatter, they would not appear in `collections.blog` and therefore not in the feed.

**Verification needed:** Check frontmatter of case study posts for correct `tags: blog` (or confirm they use the `blog/blog.json` directory data which sets the tag automatically).

**Files:** `/home/runner/work/soleur/soleur/eleventy.config.js` (lines 7-23), case study `.md` files in `/home/runner/work/soleur/soleur/plugins/soleur/docs/blog/`

### 6. Sitemap includes feed.xml as a page entry (LOW-MEDIUM)

The sitemap at `/sitemap.xml` includes `https://soleur.ai/blog/feed.xml` as a URL entry. The Atom feed is not an HTML page and should not appear in the sitemap. Search engines expect sitemap entries to be indexable HTML pages.

The sitemap template iterates over `collections.all`, which includes the feed. The feed template should set `eleventyExcludeFromCollections: true` or the sitemap should filter it out.

**File:** `/home/runner/work/soleur/soleur/plugins/soleur/docs/sitemap.njk`

---

### 7. robots.txt does not name AI-specific crawlers (LOW-MEDIUM)

The robots.txt is permissive with `User-agent: * / Allow: /`, which is sufficient -- all crawlers including AI bots (GPTBot, ClaudeBot, PerplexityBot, Google-Extended) are permitted. However, explicitly naming AI crawlers with `Allow` rules provides a stronger signal and prevents future issues if a more restrictive rule is added above the wildcard.

**File:** `/home/runner/work/soleur/soleur/plugins/soleur/docs/robots.txt`

This is a low-priority improvement. The current configuration is functional.

---

### 8. Non-blog pages lack publish/modified dates (LOW-MEDIUM)

Static pages (Getting Started, Agents, Skills, Vision, Community, Changelog) have no visible dates and no structured data indicating when they were last updated. The sitemap provides `lastmod` dates (derived from Eleventy's file date), but the pages themselves show no date information.

**Impact:** E-E-A-T guidelines favor content with visible freshness indicators. AI engines also use dates to assess content recency. Blog posts correctly display dates; static pages do not.

---

### 9. Single OG image used across all pages (LOW-MEDIUM)

Every page uses the same `og-image.png` for social sharing (`/images/og-image.png`). While this ensures consistent branding, page-specific OG images (especially for blog posts) significantly improve click-through rates on social platforms.

**File:** `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk` (line 14)

---

### 10. Heading hierarchy has minor semantic issues on homepage (LOW)

The homepage heading hierarchy is mostly correct but has one issue:
- `h3` "The Workflow" subtitle (line 91 of `index.njk`) is followed by `h3` workflow step names (Think, Plan, Build, etc.), creating sibling h3s under an h3 rather than under an h2.

The subtitle `"The Workflow"` is styled as a label but uses the `h3` semantic role via a class. This is a minor issue that does not block indexing.

Blog posts and static pages have clean heading hierarchies (h1 > h2 > h3 with no skips).

---

### 11. `<base href="/">` tag may cause issues with relative URLs (LOW)

Line 64 of `base.njk` includes `<base href="/">`. This tag changes how all relative URLs in the document are resolved. While it likely works for the current site structure, it can cause unexpected behavior with anchor links, form actions, and JavaScript `fetch` calls. Some search engine crawlers handle `<base>` differently than browsers.

**File:** `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk` (line 64)

## Recommendations (Prioritized)

### P0 -- Investigate immediately

1. **Verify meta tags render in production HTML.** Build the site locally (`npx @11ty/eleventy`) and inspect `_site/index.html` for OG tags, canonical URL, and Twitter cards. If present in build output but missing on live site, the issue is CDN/hosting-layer (check GitHub Pages build, Cloudflare, or edge workers). If absent in build output, debug the Eleventy template rendering. This single fix resolves the most impactful SEO gap on the site.

### P1 -- High impact, implement within 1-2 weeks

2. **Create an About/Author page** at `/pages/about.html` with founder bio, credentials, company mission, and social links. Update `site.json` `author.url` to point to this page. This strengthens E-E-A-T signals across all blog posts.

3. **Verify case study posts appear in Atom feed.** Check that all 4 case study markdown files in `/plugins/soleur/docs/blog/` have the correct tags or are picked up by `blog.json` directory data. If the files exist outside the `blog/` directory, the `blog.json` tag inheritance would not apply.

### P2 -- Medium impact, implement within 1 month

4. **Exclude feed.xml from sitemap.** Either add `eleventyExcludeFromCollections: true` to the feed plugin output, or add a filter in `sitemap.njk` to exclude non-HTML URLs: `{%- if not entry.url.endsWith('.xml') %}`.

5. **Add visible "Last updated" dates to static pages.** At minimum, show dates on Getting Started, Agents, and Vision pages. These are the most-visited informational pages.

6. **Add explicit AI crawler rules to robots.txt.** While `User-agent: *` is sufficient, adding named rules for `GPTBot`, `ClaudeBot`, `PerplexityBot`, and `Google-Extended` provides explicit intent.

### P3 -- Low impact, nice to have

7. **Generate page-specific OG images for blog posts.** Use a build-time image generation tool (e.g., `@11ty/eleventy-img` with SVG templates) to create unique social cards per post.

8. **Fix the "The Workflow" heading to be h2 instead of h3** on the homepage (or restyle it as a non-heading element).

9. **Add per-entry `<author>` elements to the Atom feed.** The feed currently uses only a feed-level organizational author ("Soleur"). Individual entries should include `Jean Jikig` as the author, matching the BlogPosting structured data.

10. **Consider removing the `<base href="/">` tag** and switching all internal links to absolute paths or properly relative paths. The `<base>` tag is a legacy pattern that can cause subtle issues with some crawlers and JavaScript libraries.
