### Warnings

1. **CSS loaded synchronously in `<head>` (CWV: LCP risk).** `/` ships `<link rel="stylesheet" href="css/style.css">` as a render-blocking resource. A `<link rel="preload" as="style">` is present immediately above, but the stylesheet still blocks first paint. Consider critical-CSS inlining + async load of the rest, or confirm actual LCP via Lighthouse/PSI before treating as a real problem.
2. **Hero section lacks explicit image dimensions (CWV: CLS indicator, source-level).** No `<img width height>` found on the homepage hero viewport. Current hero may be CSS/text-only (no image), in which case this is a non-issue — confirm via Lighthouse.
3. **robots.txt is permissive-by-default only.** It uses `User-agent: *` + `Allow: /` with no explicit allowlist for `GPTBot`, `ClaudeBot`, `PerplexityBot`, `Google-Extended`. Absence is sufficient for AI crawling today, but explicit `Allow` entries are a stronger defense against third-party CDN defaults (see `cq-cloudflare-block-ai-bots-not-skippable`). Verify Cloudflare "Block AI bots" feature is disabled at the zone level (Terraform `cloudflare_bot_management.ai_bots_protection = "disabled"`).
4. **`og:image` is a single shared image across all pages.** Homepage, blog posts, about, legal all point to `/images/og-image.png`. Per-page OG images would improve social CTR and LLM snapshotting. Use `gemini-imagegen` skill to generate per-post images.
5. **Plausible script (`plausible.io/js/pa-unmpBTT7YXW_UsDCGRfHH.js`) is `async` but still a third-party script on the critical network path.** Negligible for Core Web Vitals but noted for completeness.

### Recommendations

Prioritized — all optional; none block indexing.

1. **(CWV, medium)** Run Lighthouse/PageSpeed Insights on `/` and one blog post. If LCP > 2.5s, inline critical CSS for the above-the-fold section in `_includes/base.njk` and defer the rest of `css/style.css`. Source heuristics flag the sync stylesheet; a lab run confirms whether it matters.
2. **(AEO, medium)** Add explicit AI-crawler `Allow` rules to `robots.txt`:
   - `User-agent: GPTBot` → `Allow: /`
   - `User-agent: ClaudeBot` → `Allow: /`
   - `User-agent: PerplexityBot` → `Allow: /`
   - `User-agent: Google-Extended` → `Allow: /`
   - `User-agent: CCBot` → `Allow: /`
   Makes intent explicit and audit-friendly.
3. **(Zone-level, medium)** Verify Cloudflare's zone-level "Block AI bots" feature is disabled in Terraform (`cloudflare_bot_management` with `ai_bots_protection = "disabled"`). Per `cq-cloudflare-block-ai-bots-not-skippable`, this operates outside WAF and cannot be bypassed by custom rulesets. A permissive `robots.txt` is ineffective if the zone feature is on.
4. **(Social/AEO, low)** Generate per-post OG images using the `gemini-imagegen` skill. Shared `og-image.png` weakens per-post sharing and LLM visual snapshot quality.
5. **(AEO, low)** Add `BreadcrumbList` JSON-LD on blog posts (`Home → Blog → Post Title`) — helps Google render breadcrumb SERPs and gives AI models navigation context. Site is currently flat enough that this is purely upside.
6. **(E-E-A-T, low)** The Organization schema cites one NewsArticle (Inc.). As more press accrues, extend `subjectOf` with additional `NewsArticle` entries — strongest signal for AI-model trust ranking.
7. **(CWV, low)** Consider adding `font-display: swap` (or `optional`) to any self-hosted `@font-face` declarations in `css/style.css` if fonts are custom. Not verified from HTML source alone.
