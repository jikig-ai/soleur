## Critical Issues

None observed. The fundamentals — canonical URLs, JSON-LD, OG/Twitter cards, sitemap, llms.txt, AI-crawler access — are all correctly implemented in static HTML (no JS-injection risk).

## Warnings

1. **`<base href="/">` tag in `<head>` on every page.** Combined with relative anchor hrefs in the header (`href="index.html"`, `href="pricing/"`, `href="getting-started/"`), this works because of the base tag — but it is fragile. If `<base>` is ever removed, every nav link breaks. Most SEO tooling and a subset of AI crawlers handle `<base>` inconsistently. Consider switching nav anchors to absolute root-relative paths (`/pricing/`, `/getting-started/`) and removing the `<base>` tag.
2. **Soft hidden Cloudflare bot-detection link in `<body>`.** First child of `<body>` is an `<a>` to `https://www.soleur.ai/cdn-cgi/content?id=...` styled `display:none !important; visibility:hidden !important` with `rel="nofollow noopener"`. This is Cloudflare Bot Fight Mode's honeypot. It is `aria-hidden`/`nofollow`, so it is not an SEO problem in itself, but it does mean an extra DOM node + a request the browser may ping. Worth noting it exists; no action required.
3. **Plausible analytics script is `async` and blocked by CSP for hash-mismatched inline injection.** The CSP comment explicitly notes Cloudflare Bot Fight Mode injects an inline script that is intentionally blocked. Functionally fine (privacy-friendly analytics still loads), but it does mean console errors in browsers. Not visible to crawlers.
4. **No author entity on the home page Organization JSON-LD.** The home page has `Organization` with `founder: { @type: "Person", name: "Jean Deruelle" }` — but the Person entity is not given an `@id` and is not cross-linked from the `/about/` page's JSON-LD (which has only `WebPage`). Adding a `Person` entity to `/about/` JSON-LD with `@id: "https://soleur.ai/about/#jean-deruelle"` and back-referencing it from `Organization.founder` and blog `BlogPosting.author` would tighten the entity graph and improve AEO retrieval.
5. **Blog post body lacks article images / illustrations.** Every blog post has an OG image, but the article body itself is text-only (verified on `/blog/one-person-billion-dollar-company/`). Not a violation, but inline images with descriptive `alt` text would broaden multi-modal crawler coverage.
6. **Heading hierarchy on home page contains a stylistic `<h2 class="feature-grid-sublabel">The Workflow</h2>`** mid-section — semantically this is a sub-heading under the preceding h2 ("8 departments. One knowledge base..."), and would more accurately be an h3. Minor; does not break crawler parsing.
7. **No `dateModified` on home page or `/about/`.** Sitemap has `lastmod` (good), but the page-level JSON-LD does not expose `dateModified` for non-blog pages. Low priority — Google primarily relies on sitemap `lastmod` for these page types.
8. **Blog post H3 is styled at `--text-lg` (smaller than H4 expectation).** The CSS `h3{font-size:var(--text-lg)}` makes h3 visually similar to body text. Semantically correct, but visually it can feel like H2→H3 hierarchy is collapsed. Visual concern, not SEO.

## Recommendations

Prioritized — top items first.

1. **(P2) Cross-link Person entity in JSON-LD.** On `/about/`, add a `Person` node with `@id: "https://soleur.ai/about/#jean-deruelle"`, `name`, `jobTitle: "Founder of Soleur"`, `image`, `url`, `sameAs` (GitHub, X, LinkedIn). Then reference it as `{ "@id": "https://soleur.ai/about/#jean-deruelle" }` from `Organization.founder` on the home page and `BlogPosting.author` on every blog post. This gives crawlers and AI engines a single canonical entity to retrieve when answering "who founded Soleur?". File: `_includes/base.njk` (or whichever Nunjucks partial wraps the JSON-LD). Tooling: validate with [Schema Markup Validator](https://validator.schema.org/).

2. **(P3) Replace `<base href="/">` + relative nav links with absolute root-relative paths.** Edit the header partial so anchors are `/pricing/`, `/getting-started/`, etc., then drop the `<base>` tag. Reduces fragility and aligns with SEO best practice. Test by clicking through every nav link in `_site/` after build.

3. **(P3) Add `dateModified` to `WebPage` JSON-LD on static pages.** Source the value from the same `lastmod` token Eleventy already feeds into the sitemap. One-line Nunjucks change inside the JSON-LD partial.

4. **(P3) Demote `<h2 class="feature-grid-sublabel">The Workflow</h2>` on the home page to `<h3>`.** Or, alternatively, make it a true H2 by elevating its surrounding section. Current structure puts a second H2 mid-card-grid which reads like a continuation rather than a new section.

5. **(P4) Add inline images / diagrams to long-form blog posts.** Posts like `/blog/one-person-billion-dollar-company/` are body-text only. A single architecture diagram or comparison table per major section, with descriptive `alt` text and `figure`/`figcaption`, would: (a) improve dwell time, (b) feed image search, (c) give multi-modal AI engines visual context.

6. **(P4) Confirm Core Web Vitals with Lighthouse / PageSpeed Insights against the live URL.** Source-level signals look excellent (no render-blocking, font-display swap, no above-fold images), but lab scores are the source of truth. Run against `/`, `/blog/`, `/blog/one-person-billion-dollar-company/`, `/pricing/`. Capture LCP, INP, CLS for the audit record.

7. **(P5) Consider explicit `Allow:` rules per AI crawler in `robots.txt`.** Currently `User-agent: * / Allow: /` covers them implicitly (and they all return HTTP 200, confirmed). Adding explicit `User-agent: GPTBot / Allow: /` blocks for GPTBot, ClaudeBot, PerplexityBot, Google-Extended is a stronger signal and protects against future tightening. Optional — current setup is already permissive.

8. **(P5) Add an `OnlinePresenceProvider` or `Brand` entity to disambiguate "Soleur" from generic results.** Low priority; only matters if "soleur" surfaces ambiguously in AI engine results during brand monitoring.
