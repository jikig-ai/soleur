# Soleur.ai GEO/AEO Audit — 2026-05-18

Audit of soleur.ai for AI-engine consumability using the Structure / Authority / Presence (SAP) framework. Covers a sample of the most important pages: homepage, About, Vision, Pricing, Agents, Company-as-a-Service pillar, Getting Started, llms.txt, and a 27-post blog sample. Findings prioritize source citations > statistics > quotations > definitions > readability per the GEO impact ladder. Keyword-density signals are intentionally ignored (counterproductive for AI visibility).

**Live-fetch note:** WebFetch returned HTTP 526 (Cloudflare origin SSL handshake failure) for https://soleur.ai/ and https://www.soleur.ai/ at audit time. This is an infrastructure availability incident, not a content issue, and is called out separately under "Top GEO findings" because zero-availability nullifies every other AEO signal. Audit substance is graded from the Eleventy source corpus (`plugins/soleur/docs/**`), which is the deployed artifact. JSON-LD payloads, FAQ block text, citations, and author bylines are inlined in source so structural audit fidelity is preserved.

## Top GEO findings, prioritized by impact

1. **CRITICAL — Live site is HTTP 526 at audit time (P0).** `WebFetch https://soleur.ai/` and `curl -I https://soleur.ai/` both return `HTTP/2 526` from Cloudflare with the body indicating an origin SSL handshake failure between Cloudflare and the origin server. This nullifies every other AEO signal for the duration of the outage — AI crawlers (Perplexity, ChatGPT browsing, Claude.ai, Gemini) cannot index any content. Verify origin certificate validity, Cloudflare ↔ origin TLS mode, and origin server health before next audit. Hand off to the infra/seo-aeo-analyst path immediately; this audit graded the source corpus and the score above assumes the site is reachable. (Reference recent infra commit `54ce580 fix(infra): inline ACME-aware HTTPS upgrade into seo_page_redirects ruleset` — verify the redirect ruleset is not masking an origin cert issue.)
2. **Blog citation density is the largest authority gap (P1).** 27 blog posts averaging 0-1 external links per post is a citation desert relative to the marketing pages. The blog is also the largest single content surface on the site and the most likely entry point for AI-answer placement on long-tail queries. Action: minimum 3 inline citations per evergreen blog post (Anthropic primary sources, MCP spec, industry research from a16z / Gartner / HBR / McKinsey, primary-source competitor blog posts for the "Soleur vs X" series). Start with the most-trafficked posts (best-claude-code-plugins-2026, one-person-billion-dollar-company, soleur-vs-cursor).
3. **Add `last_updated` / publish-date metadata to every evergreen page (P1).** Homepage, About, Vision, Pricing, and Agents currently carry no visible publish or last-reviewed date. E-E-A-T weighting in AI ranking systems penalizes undated pages; freshness is a citation prerequisite for several AI engines (notably Perplexity). Add a `last_updated:` frontmatter field on all evergreen `.njk` pages and render it in the page header next to the H1 ("Last updated: 2026-05-18 · Author: Jean Deruelle"). Use the existing `site.statsLastVerified` pattern as the precedent.
4. **Add a stat-led summary paragraph directly under the H1 on every page (P1).** Hero slogans ("Stop hiring. Start delegating.", "Every department. One price.", "The Soleur Vision: Company-as-a-Service for the Solo Founder") are correct per brand-guide but skipped by extraction models. A 2-3 sentence factual paragraph stating "Soleur is a Company-as-a-Service platform: 66 agents, 8 departments, 67 skills, Apache-2.0 open-source, built on Claude Code and MCP. Founded 2026 by Jean Deruelle." gives AI engines a clean quote target without diluting the brand hero. Use the **General register** vocabulary from the brand guide on homepage/About/Pricing, and the **Technical register** on Vision/Agents/Getting Started.
5. **Replace "Internally called" coined-term framing with single-sentence quotable definitions on Vision (P2).** "The Global Brain", "The Decision Ledger", "The Coordination Engine" are coined terms that AI engines will not recognize as established entities. Replace each `Internally called "X"` paraphrase with a one-sentence definition the first time the term appears: "The Global Brain is Soleur's multi-model orchestration layer that selects the best model for each task — Claude for coding, GPT-4o for strategy, local models for privacy-sensitive data." Adjacent definitions are the single highest-leverage Structure fix on Vision.
6. **Diversify the press strip beyond Inc.com (P2).** The homepage "As seen in" section carries exactly one outlet. Brand mentions in AI answers depend on third-party corpus presence. Concrete actions: ship the planned Product Hunt launch (logo + link in press strip), seed reviews on GitHub Awesome lists / dev.to / HN Show, get on at least one industry analyst-firm coverage list (a16z portfolio post, Latent Space mention, ThePrimeagen / Karpathy quote tweet). Start citation monitoring: test queries weekly in ChatGPT, Perplexity, Claude.ai, Gemini for "AI agents for solo founders", "company as a service", "Soleur vs Cursor", "Claude Code plugins". Track in `knowledge-base/marketing/audits/soleur-ai/citation-monitoring.md` (file does not exist yet — create it).
7. **Source-link the authority names already used on /agents/ (P3).** Eric Evans, Michael Feathers, Dave Farley, Martin Fowler, DHH appear without links. One link per name to the canonical book / blog converts them from name-drops to citation-eligible claims. Same treatment for Karpathy, Amodei, and any other named authority that already appears unlinked in agent or skill descriptions.

Start delegating."). All audited pages comply. Recommendations #3 and #4 preserve brand voice — the stat-led summary paragraph belongs beneath the brand hero, not in place of it. Coined-term definitions on Vision (recommendation #5) should be written in the **Technical register** ("Soleur's multi-model orchestration layer that selects the best model for each task") not the General register, because Vision is a deep-content surface targeting technical builders.
