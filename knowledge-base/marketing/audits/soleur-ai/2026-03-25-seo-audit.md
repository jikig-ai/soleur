## Critical Issues

### 1. Canonical URL, OG tags, and Twitter cards not rendering in production HTML

**Severity:** Critical
**Status:** Recurring (also identified in 2026-03-23 audit)
**Impact:** Every page on the site

The `base.njk` template (lines 7-23) correctly defines:
- `<link rel="canonical" href="{{ site.url }}{{ page.url }}">`
- All OG tags: og:title, og:description, og:url, og:type, og:site_name, og:locale, og:image (with dimensions and alt)
- All Twitter card tags: twitter:card, twitter:title, twitter:description, twitter:image, twitter:site, twitter:creator

Live fetches of `https://soleur.ai` and `https://soleur.ai/blog/ai-agents-for-solo-founders/` confirm **none of these tags are present** in the served HTML. The `<title>` and `<meta name="description">` tags render correctly, as does all JSON-LD structured data. Only the OG, Twitter, and canonical tags are absent.

**Impact:**
- **Social sharing:** Facebook, LinkedIn, Twitter/X, Slack, Discord, and iMessage all display generic previews instead of rich cards with title, description, and image. For a site actively building community through Discord and social channels, this directly reduces click-through rates on shared links.
- **Canonical URLs:** Without `<link rel="canonical">`, search engines may index duplicate variants (trailing slash, www vs non-www, HTTP vs HTTPS). Google will choose a canonical itself, but it may not match the intended URL.
- **AI engines:** Perplexity, ChatGPT web browsing, and Claude's web tool use OG tags as supplementary signals for content extraction.

**Root cause hypothesis:** The tags are defined between `<meta name="description">` (line 6) and `<script type="application/ld+json">` (line 24) in the template. The description and JSON-LD render fine, but everything between them is stripped. This pattern is consistent with an HTML minifier or post-processor that strips tags it does not recognize, or a build-time issue where the Nunjucks variables evaluate to empty strings and a minifier collapses the empty tags.

**Verification:** Build the site locally with `npx @11ty/eleventy` and inspect `_site/index.html` directly. If the tags are present in the build output, the issue is at the GitHub Pages / CDN layer. If absent, debug the Eleventy rendering pipeline.

**Files:**
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk` (lines 7-23)

## Warnings

### 2. No About/Author page exists

**Severity:** Medium-High
**Impact:** E-E-A-T signals across all blog posts and the entire site

The site has no `/about/`, `/about.html`, or dedicated author profile page. The `site.json` `author.url` is set to `"https://soleur.ai"` (the homepage). Blog posts link to the author with `rel="author"` but the destination is the product homepage, not a page establishing the author's identity or credentials.

The JSON-LD `BlogPosting` schema references the author with `"url": "https://soleur.ai"`, meaning structured data also points to a non-author page.

Google's helpful content system evaluates whether content demonstrates first-hand expertise. An author link that resolves to a product homepage rather than a person's bio or credentials page weakens this signal. AI engines (Perplexity, Google AI Overviews) also use author signals to assess source quality.

**Files:**
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/_data/site.json` (line 28: `"url": "https://soleur.ai"`)
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/blog-post.njk` (line 40: author link)

### 3. Atom feed contains only 6 of 12 blog posts

**Severity:** Medium
**Impact:** RSS readers and aggregators see incomplete content

The sitemap contains 12 blog posts but the Atom feed at `/blog/feed.xml` shows only 6 entries. The feed plugin limit is 20 (`eleventy.config.js`, line 12), so the limit is not the cause.

Missing from feed:
- `case-study-brand-guide-creation`
- `case-study-business-validation`
- `case-study-competitive-intelligence`
- `case-study-legal-document-generation`
- `case-study-operations-management`
- `what-is-company-as-a-service`

The feed plugin is configured with `collection.name: "blog"`. The case study files in the `blog/` directory should inherit the `blog` tag from `blog/blog.json`. If any of these files lack the `blog` tag or are placed outside the `blog/` directory tree, they would not appear in the feed.

**Files:**
- `/home/runner/work/soleur/soleur/eleventy.config.js` (lines 7-23)
- Blog post markdown files in `/home/runner/work/soleur/soleur/plugins/soleur/docs/blog/`

### 4. Sitemap includes feed.xml as a URL entry

**Severity:** Medium-Low
**Impact:** One non-indexable URL in sitemap

The sitemap at `/sitemap.xml` contains `https://soleur.ai/blog/feed.xml`. This is an Atom XML feed, not an indexable HTML page. Search engines expect all sitemap entries to be crawlable HTML pages.

The feed is generated by the Eleventy RSS plugin, which adds it to `collections.all`. The sitemap template iterates `collections.all` without filtering.

**Files:**
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/sitemap.njk`

### 5. Static pages lack visible dates

**Severity:** Medium-Low
**Impact:** E-E-A-T freshness signals on 8 non-blog pages

Static pages (Getting Started, Agents, Skills, Pricing, Vision, Community, Changelog, Legal) display no visible publish or last-updated dates. The sitemap provides `lastmod` dates (derived from Eleventy's file date), but users and AI crawlers looking at the page itself see no date information.

Blog posts correctly display dates. Static pages do not.

### 6. Single OG image used across all pages

**Severity:** Low-Medium
**Impact:** Social sharing click-through rates

Every page uses `/images/og-image.png` for social sharing. While this ensures consistent branding, page-specific OG images (especially for blog posts with distinct topics) improve click-through rates on social platforms. The template supports per-page override via the `ogImage` frontmatter variable, but no page currently uses it.

**Files:**
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk` (line 14)

### 7. robots.txt does not name AI-specific crawlers

**Severity:** Low
**Impact:** Marginal -- current config is functional

The robots.txt uses `User-agent: * / Allow: /`, which permits all crawlers including AI bots. Explicitly naming AI crawlers (`GPTBot`, `ClaudeBot`, `PerplexityBot`, `Google-Extended`) with `Allow: /` rules provides stronger intent signaling and prevents future issues if a more restrictive wildcard rule is added.

**Files:**
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/robots.txt`

### 8. Homepage heading hierarchy has minor semantic issue

**Severity:** Low
**Impact:** Minor SEO signal

The homepage uses an h3 element for "The Workflow" (a section subtitle in `index.njk`, line 91) followed by sibling h3 elements for workflow steps (Think, Plan, Build, Review, Ship, Compound). Semantically, "The Workflow" should be an h2 so the step names (h3) nest under it. The current structure creates a flat sequence of h3 siblings, some of which are section labels and some are content items.

**Files:**
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/index.njk` (line 91)

### 9. `<base href="/">` tag present

**Severity:** Low
**Impact:** Potential subtle issues with some crawlers

Line 64 of `base.njk` includes `<base href="/">`. This changes how all relative URLs are resolved. While it works for the current site structure, some search engine crawlers handle `<base>` differently than browsers. It can also cause unexpected behavior with anchor links and form actions.

**Files:**
- `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk` (line 64)

## Recommendations (Prioritized)

### P0 -- Investigate immediately

**1. Fix meta tag rendering in production HTML.**

This is the single highest-impact SEO issue on the site and has persisted since at least 2026-03-23. The canonical URL, all OG tags, and all Twitter card tags are defined in the template but do not appear in the served HTML.

Action:
1. Build the site locally: `npx @11ty/eleventy`
2. Inspect `_site/index.html` -- search for `og:title`, `rel="canonical"`, `twitter:card`
3. If present in build output but absent on live site: the issue is at the hosting/CDN layer (GitHub Pages build pipeline, Cloudflare, or an edge worker stripping tags)
4. If absent in build output: the issue is in the Eleventy rendering pipeline -- check whether Nunjucks variables (`title`, `description`, `site.url`, `page.url`) evaluate correctly for all pages, and whether an HTML minifier or plugin is collapsing tags with empty content

Resolving this single issue would raise the Meta Tags score from 2 to 5, increasing the overall SEO score from 72 to approximately 84 (B+).

### P1 -- High impact, implement within 1-2 weeks

**2. Create an About/Author page.**

Create `/pages/about.html` (or `/pages/about.njk`) with:
- Founder name and bio
- Credentials and relevant experience
- Company mission and thesis (can reference the Vision page)
- Links to social profiles (X, LinkedIn, GitHub)
- Professional photo (optional but recommended)

Update `/plugins/soleur/docs/_data/site.json` to set `author.url` to `"https://soleur.ai/pages/about.html"`. This propagates to all blog posts' author links and BlogPosting schema automatically.

**3. Fix Atom feed to include all blog posts.**

Verify that all 12 blog post markdown files in `/plugins/soleur/docs/blog/` are tagged correctly. The `blog.json` directory data file sets `"tags": "blog"` for all files in the directory. If any case study files have their own frontmatter `tags` that override (rather than merge with) the directory data, they would lose the `blog` tag. Check each case study file's frontmatter for a `tags` key.

### P2 -- Medium impact, implement within 1 month

**4. Exclude feed.xml from sitemap.**

Add a URL filter to the sitemap template to exclude non-HTML entries:

In `/plugins/soleur/docs/sitemap.njk`, change:
```
{%- for entry in collections.all %}
```
To:
```
{%- for entry in collections.all %}{%- if not entry.url.endsWith('.xml') %}
```
And add `{%- endif %}` before `{%- endfor %}`.

**5. Add visible "Last updated" dates to key static pages.**

At minimum, add a date display to Getting Started, Agents, Skills, and Vision pages. These are the highest-traffic informational pages. Use the existing `readableDate` filter from the Eleventy config.

**6. Add explicit AI crawler rules to robots.txt.**

Append named rules for AI crawlers after the existing wildcard rule:

```
User-agent: GPTBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: Google-Extended
Allow: /
```

### P3 -- Low impact, nice to have

**7. Generate page-specific OG images for blog posts.** The template already supports per-page `ogImage` frontmatter. Create distinct social card images for at least the comparison posts (Soleur vs. Cursor, Soleur vs. Cowork, Soleur vs. Notion) which are likely shared most frequently.

**8. Promote "The Workflow" heading to h2 on homepage.** Change line 91 of `index.njk` from `<h3 class="feature-grid-sublabel">The Workflow</h3>` to `<h2 class="feature-grid-sublabel">The Workflow</h2>` and adjust CSS if needed.

**9. Consider removing the `<base href="/">` tag.** Convert internal links to use absolute paths or properly relative paths. This eliminates a potential source of subtle crawling issues.

**10. Add per-entry author elements to the Atom feed.** The feed currently uses only a feed-level author ("Soleur"). Individual entries should include the actual author name, matching the BlogPosting schema.
