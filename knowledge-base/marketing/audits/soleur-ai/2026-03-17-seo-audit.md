# SEO/AEO Audit Report

**Site:** Soleur | **URL:** https://soleur.ai | **Pages:** ~25 (7 static pages + 8 blog posts + 9 legal pages + index)

### Critical Issues

**1. robots.txt does not explicitly allow AI crawlers (Medium-High)**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/robots.txt`

The current robots.txt is:

```
User-agent: *
Allow: /
Sitemap: https://soleur.ai/sitemap.xml
```

While the wildcard `User-agent: *` with `Allow: /` is permissive and does not block any crawler, the file does not include explicit `User-agent` directives for major AI crawlers. Some AI crawlers (GPTBot, PerplexityBot, ClaudeBot, Google-Extended) may default to restrictive behavior unless they see an explicit allow. This is a low-risk issue since the wildcard rule covers them, but explicit entries would be a defensive improvement. No blocking is present -- this is not a true critical issue.

**Verdict: PASS (with a minor recommendation below).**

**2. No `updated` (dateModified) field on any blog post**

Files: All 8 blog posts under `/home/runner/work/soleur/soleur/plugins/soleur/docs/blog/`

The `blog-post.njk` template at `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/blog-post.njk` line 16 supports `dateModified`:

```nunjucks
"dateModified": "{{ updated | dateToRfc3339 if updated else date | dateToRfc3339 }}",
```

But no blog post defines an `updated` frontmatter field. This means `dateModified` always equals `datePublished` in the BlogPosting JSON-LD. Google uses `dateModified` to assess content freshness. When the two dates are identical, Google has no signal that content has been reviewed or refreshed. For evergreen content like "What Is Company-as-a-Service?" this is a missed opportunity.

**3. Non-page templates included in sitemap (articles.njk redirect page)**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/sitemap.njk`

The sitemap iterates `collections.all`:

```nunjucks
{%- for entry in collections.all %}
```

The `articles.njk` redirect page uses `eleventyExcludeFromCollections: true`, so it is correctly excluded. The `404.njk`, `sitemap.njk`, and `llms.txt.njk` are also correctly excluded. However, the sitemap does not filter out pages that might be thin or non-indexable. Currently this is working correctly.

**Verdict: PASS.**

**4. Blog posts lack visible `dateModified` display for users**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/blog-post.njk` (lines 38-40)

Only `datePublished` is shown to users:

```html
<time datetime="{{ date | dateToRfc3339 }}">{{ date | readableDate }}</time>
```

There is no "Last updated" display. Google's helpful content system rewards pages that show when content was last reviewed, particularly for YMYL (Your Money or Your Life) topics -- and blog posts about business validation and legal document generation touch YMYL territory.

### Warnings

**5. OG image is hardcoded to a single generic image across all pages**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk` (line 14)

```html
<meta property="og:image" content="{{ site.url }}/images/og-image.png">
```

Every page (homepage, blog posts, agents page, skills page) shares the same OG image. When blog posts are shared on social media, they all look identical. Page-specific OG images (especially for blog posts) significantly improve click-through rates from social shares and AI citation previews.

**7. Heading hierarchy issue on homepage: h3 used inside feature grid before proper h2 context**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/index.njk` (line 91)

```html
<h3 class="feature-grid-sublabel">The Workflow</h3>
```

This `h3` appears after a `div.feature-grid-departments` which contains its own `h3` elements. The heading hierarchy jumps: h2 ("Agentic engineering from idea to shipped") -> h3 (department names) -> h3 ("The Workflow") -> h3 (workflow step names). While `h3` -> `h3` is technically valid sibling usage, "The Workflow" functions semantically as a sub-section that should arguably be an `h2` or a distinct heading level to distinguish it from the department card headings above it. This is a minor heading semantics issue.

**9. Non-page-specific OG image for blog posts in BlogPosting JSON-LD**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/blog-post.njk` (line 18)

```json
"image": "{{ site.url }}/images/og-image.png",
```

The BlogPosting structured data references the same generic site-wide image for every blog post. Google recommends that the `image` property be specific and representative of the article content.

**10. Sitemap lacks `changefreq` and `priority` values**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/sitemap.njk`

```xml
<url>
  <loc>{{ site.url }}{{ entry.url }}</loc>
  <lastmod>{{ entry.date | dateToShort }}</lastmod>
</url>
```

While Google has stated they largely ignore `changefreq` and `priority`, other search engines (Bing, Yandex) still use them. More importantly, the `lastmod` date uses `entry.date` which for many pages is the file creation date (from `page.date` in Eleventy), not a meaningful content modification date. This may not accurately reflect when content was last updated.

**11. Navigation link URLs use relative paths without leading slash**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_data/site.json` (lines 16-23)

```json
{ "label": "Get Started", "url": "pages/getting-started.html" },
```

Navigation URLs lack a leading `/`. The site uses `<base href="/">` in the `<head>` (line 64 of `base.njk`), so these resolve correctly. However, the `<base>` tag can cause unexpected behavior with anchor links and JavaScript URL resolution. This is a fragile pattern.

**12. No `hreflang` tag for international targeting**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk`

The site is published in English but operated by a French company (Jikigai, Paris). There is no `hreflang` attribute. Since the site appears to be English-only, this is minor, but if the company plans to serve French or EU users, `hreflang` should be added.

**13. Agents page has nested h3 headings inside catalog grid**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/pages/agents.njk` (line 53)

```html
<h3 class="card-title">{{ agent.name }}</h3>
```

Agent cards inside subcategories already have an `h3` for the subcategory name (line 45), and then each card also uses `h3` for the agent name. This creates sibling `h3` elements under an `h3`, which is technically flat rather than hierarchical. The agent names should arguably be `h4` to maintain strict heading hierarchy.

**14. Vision page has no meta description differentiation from homepage**

File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/pages/vision.njk` (line 3)

```yaml
description: "Where Soleur is headed. The Company-as-a-Service platform that gives solo founders the leverage of a full organization."
```

This is fine as-is, but the description does not mention any specific vision content (milestones, master plan, revenue philosophy). A more specific description would improve SERP snippet quality.

### Recommendations

Ordered by estimated SEO/AEO impact:

1. **Add `updated` frontmatter to blog posts that have been revised.** Add an `updated: YYYY-MM-DD` field to any blog post that has been meaningfully edited since publication. This populates the `dateModified` field in BlogPosting JSON-LD and signals content freshness to Google.
   - Files to edit: All 8 blog post `.md` files under `/home/runner/work/soleur/soleur/plugins/soleur/docs/blog/`
   - Also display "Last updated" in the blog post template at `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/blog-post.njk`

2. **Add FAQPage JSON-LD to "Why Most Agentic Tools Plateau."** This blog post has three FAQ items (lines 145-157) formatted as `### ` headings but no corresponding FAQPage structured data. The other two major blog posts already have this markup.
   - File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/blog/why-most-agentic-tools-plateau.md`

3. **Add page-specific OG images for blog posts.** Create unique social share images for at least the three pillar blog posts (What Is CaaS, Why Most Tools Plateau, Soleur vs. Cowork). Update the `blog-post.njk` template to support an optional `ogImage` frontmatter field that falls back to the default.
   - Files: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/blog-post.njk`, `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk`

**Priority 2 (Medium Impact)**

4. **Fix heading hierarchy on the agents page.** Change agent card titles from `h3` to `h4` when they appear inside subcategories that already use `h3` for the subcategory name.
   - File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/pages/agents.njk` (line 53)

5. **Add explicit AI crawler user-agent entries to robots.txt.** While the wildcard covers them, adding explicit entries is a signal of intentional openness to AI indexing:

   - File: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_includes/base.njk`
   - Also update: `/home/runner/work/soleur/soleur/plugins/soleur/docs/_data/site.json` nav URLs to use leading `/`

11. **Consider adding `SearchAction` to the WebSite schema** if site search is ever implemented. This enables Google's sitelinks search box.

12. **Run Lighthouse / PageSpeed Insights** on the live site for actual Core Web Vitals lab measurements. The source-level indicators are all positive (minimal JS, no frameworks, font-display: swap, no images in content), but real-world measurements may reveal issues not visible in source analysis (CDN latency, server response time, Cloudflare processing).

### Summary

The site has a strong SEO/AEO foundation. Structured data is well-implemented across the homepage, blog posts, and FAQ pages. Meta tags (OG, Twitter, canonical) are correctly templated with dynamic values. The llms.txt follows the specification. The robots.txt is permissive. The sitemap is collection-driven. Content is fully rendered at build time (no JS-only rendering). E-E-A-T signals are present through author attribution, dates, legal documents, and cited external sources.

The main gaps are: (1) no `dateModified` signals on blog content, (2) missing FAQPage structured data on one blog post, (3) a single generic OG image across all pages, and (4) heading hierarchy inconsistencies on the agents page. All are fixable with targeted template edits.
