### Critical Issues

None. No findings that block search indexing or AI discoverability.

### Warnings

1. **robots.txt does not explicitly name AI crawlers.** The file uses `User-agent: *` + `Allow: /`, which is sufficient (absence of a rule allows a bot by default), but an explicit allow block listing GPTBot, ClaudeBot, PerplexityBot, Google-Extended, CCBot, and Applebot-Extended would signal intent clearly and protect against a future CDN policy flip. Severity: low.
2. **BlogPosting `image` property uses the site-wide OG image, not the per-post image.** The blog post has a per-post OG image (`/images/blog/og-soleur-vs-cursor.png`) in meta tags but the JSON-LD Article schema references the generic `/images/og-image.png`. Google uses the schema image for rich results. Severity: medium.
3. **`og:image:alt` on blog posts is generic** ("Soleur - Stop Hiring. Start Delegating.") rather than describing the per-post image. Minor impact on social shares and accessibility. Severity: low.
4. **No `llms-full.txt` companion file.** The spec-compliant `llms.txt` links to key docs but there is no full-content aggregation file for LLMs that want the entire corpus in one fetch. Nice-to-have, not required. Severity: low.
5. **CSS loads synchronously in `<head>`.** `<link rel="preload">` + `<link rel="stylesheet">` pattern is in place, but the stylesheet still blocks first paint. Inlining critical-path CSS or moving to `rel="preload" ... onload="this.rel='stylesheet'"` would reduce LCP. Severity: low.

### Passed Checks

- **Structured data present in static HTML** (not JS-injected): WebSite, WebPage, SoftwareApplication with offers/pricing, BlogPosting with author, FAQPage on both homepage and blog post.
- **Canonical URL** set on every page, absolute, matching the page URL.
- **Open Graph tags complete**: `og:title`, `og:description`, `og:url`, `og:type` (website/article), `og:site_name`, `og:locale` (`en_US`), `og:image` with width/height/alt.
- **Twitter/X card complete**: `summary_large_image`, title, description, image, image alt, site handle, creator handle.
- **Page-specific titles and descriptions** (not hardcoded).
- **Sitemap valid**: 36 URLs, every entry has `<lastmod>`, uses collections (dates reflect content updates).
- **llms.txt spec-compliant**: H1 title, blockquote tagline, descriptive paragraphs, `## Use Cases`, `## Docs`, `## Optional` sections with linked resources.
- **robots.txt permissive**: `Allow: /` for all user-agents; sitemap reference included.
- **E-E-A-T signals strong**: Named author ("Jean Deruelle, Founder of Soleur") with `rel="author"` link to /about/, visible `<time datetime>` element showing publish date, Person schema with `jobTitle`, `article:published_time` meta tag, `dateModified` in Article JSON-LD, dedicated /about/ page with author bio.
- **Heading hierarchy clean**: one H1 per page, proper H2 > H3 nesting on homepage and blog, no level skips.
- **HTTPS** with CSP header (`upgrade-insecure-requests`).
- **Analytics loaded async** (Plausible, `<script async>`).
- **CSS preloaded** via `<link rel="preload" as="style">`.
- **No images on homepage** → no LCP image concerns, no CLS from unsized images.
- **Atom feed advertised** for blog (`<link rel="alternate" type="application/atom+xml">`).
- **charset, viewport, favicon** all present.

### Recommendations

Ordered by impact-to-effort ratio:

1. **[P2] Use per-post image in BlogPosting JSON-LD.** In the blog template, set `"image"` in the Article schema to the same value as `og:image` (which is already per-post). One-line change in the Eleventy blog layout.
2. **[P2] Set per-post `og:image:alt`.** Replace the hardcoded `"Soleur - Stop Hiring. Start Delegating."` with the post title or a short per-post description. Again a blog-layout template edit.
3. **[P3] Add explicit AI-crawler allow rules in robots.txt.** Append stanzas for `GPTBot`, `ClaudeBot`, `PerplexityBot`, `Google-Extended`, `CCBot`, `Applebot-Extended` with `Allow: /`. Defensive signal even though the wildcard already covers them.
4. **[P3] Generate `llms-full.txt`** aggregating the full content of `/getting-started/`, `/agents/`, `/skills/`, `/vision/` into a single markdown file, permalinked at the site root. Improves discoverability for LLMs that prefer whole-corpus ingestion.
5. **[P3] Inline critical CSS** for above-the-fold content or load `css/style.css` non-blocking via preload+swap. Measure impact with Lighthouse first — current LCP may already be fine given the text-only homepage.
6. **[P3] Consider adding `Organization` schema** at site level with `sameAs` pointing to GitHub, X/Twitter, Discord. Would improve knowledge graph entity resolution. Skip if brand-authority score is already healthy.

## Key Findings

- **[P2] BlogPosting schema `image` is not per-post** — blocks rich image thumbnails in Google Discover/Search; fix in blog layout template.
- **[P2] `og:image:alt` is hardcoded across all blog posts** — reduces accessibility and social-share quality.
- **[P3] robots.txt uses wildcard only** — works today, but an explicit AI-bot allowlist is a belt-and-suspenders improvement.
- **[P3] Missing `llms-full.txt`** — nice-to-have for AI corpus ingestion; the spec-compliant `llms.txt` is already in place.
- **[P3] CSS is render-blocking** — minor LCP hit; validate with real Lighthouse data before optimizing.
