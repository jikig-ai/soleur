## Discord

Two very different shapes of Claude Code extension keep getting compared on the wrong axis.

One is a portable skill library -- a catalog of self-contained skills you pick and mix. The canonical exemplar is alirezarezvani/claude-skills.

The other is a workflow plugin -- opinionated orchestration across a brainstorm → plan → work → review → compound lifecycle, with a compounding knowledge base that every agent reads from and writes to. Soleur is the canonical exemplar of this shape.

These are different categories. A library answers "what's the next capability I can drop in?" A workflow plugin answers "how do I run every department without a team?"

For a solo founder, the answer is usually the workflow plugin -- with library skills layered on top where the gaps appear.

**Body:**

When I started looking at Claude Code extensions, the comparison everyone defaulted to was head-count. The bigger catalog wins, right? I spent some time on that framing and concluded it misses the actual question.

There are two different shapes of Claude Code extension, and they answer different questions:

**Portable skill library.** A catalog of self-contained skills. Breadth-first, unopinionated, pick-and-mix. The reference implementation is alirezarezvani/claude-skills -- a broad catalog of independent skills you can drop into any Claude Code install. Orchestration is your job; the library hands you the pieces.

**Workflow plugin.** Opinionated orchestration across a brainstorm → plan → work → review → compound lifecycle, with a compounding knowledge base that every agent reads from and writes to. Soleur is the canonical exemplar. Decisions in session 1 shape what session 50 produces. Cross-domain coherence is the default, not a custom integration.

For a solo founder running every department alone -- marketing, legal, finance, ops, sales, support -- the workflow-plugin shape is the substitute for the team you didn't hire. Library skills layer on top where the lifecycle doesn't cover something.

The category-creation point is the one that took me longest to internalize: these aren't on the same leaderboard. The right question is "which shape does what I need this week?" Sometimes both.

**Body:**

Spent the last month looking at how solo founders are evaluating Claude Code extensions. The comparison everyone defaults to is head-count, which misses the actual question.

There are two different shapes:

- **Portable skill library** (alirezarezvani/claude-skills is the canonical exemplar): a catalog of self-contained skills, breadth-first coverage, unopinionated about orchestration. You pick what you need and drop it into your existing workflow.

- **Workflow plugin** (Soleur is the canonical exemplar): opinionated orchestration across a brainstorm → plan → work → review → compound lifecycle, with a compounding knowledge base every agent reads from and writes to. The plugin runs an organization, not a tool.

Library skills compose horizontally -- more skills, more coverage. Workflow plugins compose vertically -- the lifecycle stack, the knowledge that compounds across sessions, the cross-domain coherence between marketing, legal, ops, finance.

Solo founders running every department without a team typically need the workflow-plugin shape as the substrate, with library skills layered on top to fill specific capability gaps.
