# ADR-068: Validation-gated classifier-skill edits via a delimited-block projection

- **Status:** Accepted
- **Date:** 2026-06-29
- **Issue:** #5702 / PR #5701 (feat-skill-eval-gate)
- **Lineage:** eval-harness skill (promptfoo grid + `measure-classification.cjs`); SkillOpt's
  validation-gated acceptance + rejected-edit buffer (genetic/Pareto evolution and autonomous
  self-editing deliberately NOT adopted — NG1–NG3 in the brainstorm); #5703 (deferred CI backstop),
  #5704 (deferred broader gated-skill catalog).

## Context

Soleur's two highest-traffic classifiers — the `/go` routing table
(`plugins/soleur/commands/go.md`) and the ticket-triage P-level rubric
(`plugins/soleur/agents/support/ticket-triage.md`) — are edited in-session by `heal-skill` (the
primary path that rewrites classifier rules), occasionally by `compound`, and by manual / one-shot
edits. Before this change, such an edit was applied unverified: nothing measured whether the new
prose actually classifies better, or whether it silently regressed other cases.

The eval-harness already measured a skill-arm-vs-baseline delta, but its skill-arm prompts were a
**hand-distilled paraphrase** of the production prose, with no mechanical link. A plan-review (Kieran
P1-1) showed this makes a gate dishonest: it would measure stale paraphrase text, not the edit under
test — a false-accept surface. The harness also bundled verdict logic with the live promptfoo run,
so the accept/reject rule was untestable without spending API.

## Decision

Classifier-skill rules live in a **delimited block** — wrapped in HTML-comment sentinels
(`<!-- eval-gate:block:<id>:start -->` … `:end`) — that is the **single source of truth** for both
production and the eval projection. Edits to that block are **validation-gated** before they are
applied, **proposer-agnostic** (heal-skill, compound, or a manual/one-shot edit all route through the
same gate):

1. **Mechanical projection.** The eval-harness skill-arm prompt is generated from the block by
   `extract-block.cjs` + `gen-skill-prompt.cjs` — never hand-copied. Regenerated on every source-block
   edit; an AC4 round-trip test asserts committed == generated byte-for-byte.
2. **Pure verdict.** `verdict.cjs` `computeVerdict()` is pure (no I/O, no shell-out, no LLM). The
   thin orchestrator `eval-gate.cjs` shells promptfoo **skill-arm-only** (current block vs candidate
   block — the baseline control arm is irrelevant to a regression gate), normalizes the JSON, and
   calls the verdict. Accept iff the corpus does not regress
   (`candidate_rate ≥ current_rate − ε`, ε = one-task-equivalent, boundary inclusive) **and** the
   targeted case passes (pooled candidate rate ≥ 0.5).
3. **Fail-closed.** Any gate error (missing key, promptfoo non-zero, malformed task) exits non-zero
   and defaults to NOT accept — the edit is not applied.
4. **Synthesized-only corpus growth.** `--append-on-accept` adds the targeted golden task to the
   corpus only on accept, and rejects real-data-shaped input (`cq-test-fixtures-synthesized-only`).

**Decision PRECONDITION (per Kieran P3-4):** the gate is only honest while the skill-arm prompt is a
**mechanical projection** of the block. If anyone reverts the prompts to hand-copied snapshots, the
gate silently no-ops on out-of-block edits (the exact P1-1 failure). The AC4 round-trip test and the
deferred #5703 CI projection-integrity check guard this precondition.

## Alternatives Considered

- **Genetic / Pareto evolution of skill prose (GEPA / EvoSkill).** Rejected — multi-candidate
  population search is far beyond a no-regression acceptance gate; high API cost, no demand.
- **Autonomous unattended optimizer (SkillOpt-style background self-editing).** Rejected — Soleur
  does not self-edit classifier rules unattended; in `HEADLESS_MODE` proposers skip-gate-and-defer
  rather than spend unattended, and #5703 catches at CI.
- **CI-only gate (no in-session hook).** Deferred to #5703, not chosen as the v1 mechanism — the
  primary value is blocking a bad edit *before* it is applied in-session; CI is the proposer-agnostic
  backstop for manual/one-shot edits, complementary rather than a replacement.

## C4 impact

Component-view only. Adds `evalharness` component to the Soleur Plugin block in `model.c4` with the
edge `compound -> evalharness "Validates classifier-skill edits before applying"`, and includes
`platform.plugin.evalharness` in the `views.c4` component view. No external actor, vendor edge, or
data store enters the model — this is an internal dev-tooling loop over synthesized fixtures.
(`heal-skill` is the primary proposer but is not modeled as a C4 component, so no edge is invented
for it.)

## Consequences

- **Positive:** classifier-rule edits are empirically gated (no-regression + target-passes) before
  applying, proposer-agnostic; the production block is the SSOT for both runtime and eval, killing
  the fixture-sync drift class; the verdict is pure and unit-tested with zero API.
- **Negative / watch:** the projection link is now a standing invariant (the AC4 test + #5703 guard
  it); a too-strict gate can block a good fix (recoverable in-session); the v1 rejected-edit buffer
  is local-per-machine (cross-operator buffer deferred).
